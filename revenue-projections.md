### Revenue

| Assumption | Value | Basis |
|-----------|-------|-------|
| Monthly ARPU | ₹350 | Blended: (₹499×40% + ₹250×45% + ₹400×15%) |
| Monthly churn | 8% | Dating app avg: 7-12% |
| Payment success rate | 92% | UPI has 95%+ success, cards 85% |
| Refund rate | 2% | Within 48h window |

### Why ₹350 ARPU (not ₹499)?

```
ARPU = (Monthly share × ₹499) + (Yearly share × ₹250*) + (Family share × ₹400**)
     = (0.40 × ₹499) + (0.45 × ₹250) + (0.15 × ₹400)
     = ₹200 + ₹112 + ₹60
     = ₹372 ≈ ₹350 (rounded conservatively)

* ₹2,999/yr ÷ 12 = ₹250/mo effective
** ₹799/mo ÷ 2 profiles = ₹400 revenue per "unit"
```

## 5. Unit Economics

### Lifetime Value (LTV)

```
LTV = ARPU / Churn Rate
    = ₹350 / 0.08
    = ₹4,375 per paying user
    = ~$52
```

### Customer Acquisition Cost (CAC)

| Channel | Y1 Budget | Users Acquired | CAC |
|---------|-----------|---------------|-----|
| Organic (word of mouth) | ₹0 | 34,260 | ₹0 |
| Referral program | ₹0 (premium days as reward) | 14,275 | ₹0* |
| Campus ambassadors | ₹50,000 (swag, events) | 5,710 | ₹9 |
| Content/SEO | ₹25,000 (hosting, tools) | 2,855 | ₹9 |
| **Blended CAC** | **₹75,000** | **57,100** | **₹1.3** |

*Referral reward cost is premium days, not cash. Effective cost ≈ ₹15/user in server costs.

### LTV:CAC Ratio

```
LTV:CAC = ₹4,375 / ₹1.3 = 3,365:1 (extremely high because Y1 is organic-only)
```

**Realistic adjusted CAC** (if we include opportunity cost of founder time): ~₹100-200/user → LTV:CAC = 22:1 to 44:1 (still excellent).

### Payback Period

```
Payback = CAC / (ARPU × Gross Margin)
        = ₹200 / (₹350 × 0.85)
        = 0.67 months
        ≈ 20 days
```

## 9. Path to Profitability

### Break-Even Analysis

```
Monthly fixed costs: ₹13,500
ARPU: ₹350
Gross margin per user: ₹350 × 0.946 = ₹331
Break-even paying users: ₹13,500 / ₹331 = 41 users
Break-even MAU: 41 / 0.035 = 1,171 MAU
```

**We break even at ~1,200 MAU** — expected by Month 4.

## 10. Key Risks & Mitigations

| # | Risk | Impact | Probability | Mitigation |
|---|------|--------|-------------|------------|
| 1 | **Low conversion rate** (<2%) | Revenue 50% below target | Medium | A/B test pricing; improve trial experience; add more premium features |
| 2 | **High churn** (>12%) | LTV drops below ₹3,000 | Medium | Improve match quality; add engagement features; win-back campaigns |
| 3 | **Payment failures** (UPI issues) | Lost revenue at checkout | Low | Multi-method fallback; retry logic; webhook as backup |
| 4 | **Competitor response** (Shaadi drops prices) | Pressure on pricing | Medium | Differentiate on safety + family features (not price alone) |
| 5 | **Regulatory risk** (DPDP enforcement) | Compliance costs | Low | Already DPDP-compliant by design |
| 6 | **Slow user growth** | Delayed revenue milestones | Medium | Double down on referral program + campus ambassadors |
| 7 | **Gender imbalance** | Low match rate → churn | High | Safety features attract women; "Respectful Initiation" mode |
| 8 | **Firebase cost spike** | Margin compression | Low | Optimize queries; implement caching; migrate to self-hosted if needed |
